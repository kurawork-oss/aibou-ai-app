# THE FORGE OS — 進化ロードマップ（EVOLUTION PLAN）

> 2026-07-13 策定。8ドメイン×10案（計80案）を実コードベースで洗い出し、
> 重複統合・価値×実現性で厳選したもの。工数目安: S=数時間 / M=1日 / L=数日。
> すべて現行スタック（Next.js + FastAPI + Gemini + Supabase / 無料枠中心）で実装可能。

---

## 🌟 目玉進化 TOP5（このOSを象徴する進化）

### 1. 1. リアルタイム音声会話モード×オーディオリアクティブ・コア 
無音検出で回るハンズフリー会話ループと、実際の声・TTS出力にCoreOrbが波打つ演出の統合。既存部品（Web Speech/edge-tts/CoreOrb）の組み合わせだけで「JARVISに話しかける」体験を無料枠で完成させる、このOSの世界観を象徴する進化。

### 2. 2. WEBアプリ生成＋即時プレビュー 
「作って」の3秒後に右ペインで動くアプリが立ち上がるv0/Claude Artifacts級の体験。プロンプト1種とiframe srcDocで実現でき、FORGEという名の核心価値を一段引き上げる看板機能。

### 3. 3. モーニングブリーフィング＋AIデイリープラン 
毎朝開くと今日の予定・期限・通知を要約して読み上げ、1日をタイムボックス化して提案・一括登録。「毎日必ず開く理由」を作る起点儀式で、タスク・予定・通知の既存資産を1つの体験に束ねる。

### 4. 4. Vault出典引用付き回答 
回答の主張ごとに引用チップ→タップで原文該当箇所へジャンプ。NotebookLM最大の差別化要素を自分の資料庫で再現し、AI回答の信頼性を別次元にする。docsのタイトル照合だけで埋め込み不要という実現性の高さも魅力。

### 5. 5. モード別テーマカラー・システム 
モード切替のたびにコア・星空・地平線・UIの色温度がCSS変数連動で変わる「生きているOS」演出。個別機能ではなく全モードの体験を貫く空間アイデンティティとして、このアプリの独自性を最も安く最大化する。

## ⚡ Quick Wins — 即効性が高い改善（S/M工数・14件）

### 1. 全モード共通のMarkdown＋コードハイライト描画 （M）
Chat・Vault・Forge(DOC)の生テキスト表示をreact-markdown＋highlight.jsに統一置換。コードブロックの言語ラベル＋ワンクリックコピー、表・リスト整形、ストリーミング中の逐次パースに対応し読み味が一新される。3ドメインの重複案を統合した最優先の即効改善。

### 2. メッセージ編集→再送＋任意ターン再生成 （S）
ユーザーバブルに編集ボタンを追加し、その場で修正→以降を切り捨てて再ストリーミング。再生成も最終ターン限定から任意ターンに拡張。regenerate()流用でフロントのみで完結。

### 3. 会話の横断検索＋AI自動タイトル （S）
履歴パネルに本文込みのインクリメンタル検索とヒット箇所ジャンプを追加。タイトルは先頭30文字方式をやめ初回応答時にgemini-flashで要約生成し、リネーム・ピン留めにも対応。

### 4. ワンタップ完了チェックボックス＋Undo （S）
タスクカード左端の丸チェックで即完了（現状は展開→5択の2〜3タップ）。ストライク演出と5秒の「元に戻す」トーストで誤タップを救済。既存updateTask APIのみで実装可能。

### 5. Renderコールドスタート緩和 （S）
GitHub Actionsの5分cronで/healthをping、さらにEntryGate表示時に先行fetchしBootScreen中にバックエンドを起こす。起床検知時は「REACTOR WARMING UP」演出＋自動リトライで50秒無言を解消。毎回の起動体験に直結。

### 6. クロスフォージ変換 （S）
生成結果ツールバーに「→SLIDES」「→DOC」「→IMAGE」ボタンを追加し、成果物本文をコンテキストに別種で再生成。企画書→スライド→キービジュアルと成果物が連鎖するワークフローになる。既存forgeGenerate呼び出しの前置だけ。

### 7. SLIDESフルスクリーン発表モード （M）
生成Markdownを---で分割しテーマ付きスライドデッキとしてレンダリング。矢印/スワイプ送り・Fullscreen API・プログレスバー付きで、「Markdownをpre表示」から「そのまま発表できる」に変わる。

### 8. Mermaid図のインライン描画＋SVG保存 （S）
mermaid.liveへの貼り付け頼みだった図解をmermaidパッケージでVault内に直接描画。パース失敗時はGeminiに1回だけ自動修正させるリトライ付き、SVGダウンロードも追加して外部サイト往復を消す。

### 9. モード別テーマカラー・システム （M）
CHAT=シアン、FORGE=オレンジ、VAULT=紫などモードごとのアクセントHueをCSS変数遷移で切替え、CoreOrb・星空・地平線グローも同変数に連動。モード切替のたびに空間全体の色温度が変わり「生きているOS」感が増す。

### 10. カスタムAIの実戦投入 （M）
run_workflowがスキーマにありながら無視しているsteps[].ai_idを修正し、ステップごとにpersona/rules/モデルを適用。AIカードに「このAIとChatで話す」導線も追加し、Studioで作ったAIが飾りでなくなる最重要修正。

### 11. 変数マッピング＋ステップ単体テスト （S）
{input}に加え{step1}{step2}{date}等の参照を全ステップのテンプレートで可能にし、ビルダーに変数チップと各ステップの「▶テスト」実行（前段出力の自動引き継ぎ付き）を追加。多段フローの表現力とデバッグ体験が激変する。

### 12. 承認キューのリッチプレビュー＋インライン編集 （M）
タイトル1行だけの承認キューをタグ・説明文・記事Markdownの全文プレビューに拡張し、その場でタイトルやタグを直してPATCH保存。毎朝の「見て・直して・承認」が1画面で完結する。

### 13. 却下→修正指示つき再生成 （S）
却下時に「タイトルが硬い」等の一言指示を添えると、元payloadと指示をGeminiに渡して同テーマを再生成しpendingへ積み直す。ゼロからやり直さず差分改善で回せるため承認率が上がり朝の作業が軽くなる。

### 14. 収益実績ダッシュボード （M）
income_recordsテーブルを追加し媒体別（Shutterstock/YouTube/note）の売上・PV・DLを手入力/CSV貼り付けで記録。月次推移グラフと承認→収益化の転換率を表示し、どのテーマが実際に稼いだかをジョブと紐付けて可視化する。

## 🚀 Big Bets — 体験を一段引き上げる大型案（10件）

### 1. リアルタイム音声会話モード（音に反応するコア） （L）
既存の音声認識とTTSをループ化した全画面会話モード。無音1.5秒で自動送信→ストリーミング完了と同時にTTS再生→再生終了でマイク再開、発話割り込みにも対応。Web Audio Analyserでマイク入力とTTS出力にCoreOrbが実際に波打つ演出を統合し、無料枠だけでGemini Live級の体験を作る。

### 2. WEBアプリ生成＋iframe即時プレビュー （M）
Forgeに単一ファイルHTML/JS/CSSアプリを生成するWEBタブを追加し、sandbox付きiframe(srcDoc)で右ペインに即実行表示。PREVIEW/CODEトグル付きで「生成3秒後に動くアプリが見える」v0/Artifacts級の体験になり、DL必須のStreamlitとの差が埋まる。

### 3. Vault出典引用付き回答＋Q&Aスレッド化 （L）
回答の各主張に[資料タイトル]引用を付けさせてチップ化し、タップで原文該当箇所へスライドイン表示。あわせて未使用のchat jsonb列で質問応答をノートブックごとに永続化し、「それをもっと詳しく」のフォローアップを可能にする。NotebookLMの中核体験を移植する統合案。

### 4. VaultのURL・PDF取込 （L）
URL貼り付けでhttpx＋trafilaturaの本文抽出、PDFはpypdfでテキスト化してdocsへ追加。ドロップゾーンに.pdfを許可しURLインポート欄を新設。TXT/MD手貼りのみの現状から実用的な資料収集ツールに変わる。

### 5. 会話履歴のSupabase同期（クロスデバイス） （L）
localStorage専用の会話履歴を既存KeychainのFernet暗号化パターンで暗号化しSupabaseにも保存、スマホPWA⇄PCで同じ会話を継続できる。オフライン時はlocalStorageにフォールバックし再接続時にマージ。PWA常用の前提となる同期基盤。

### 6. タスクの期限・優先度＋今日ビュー（自然言語クイック追加） （L）
tasksにdue_date/priority列を追加し、「金曜までに請求書送付 !高」をGeminiで構造化するクイック追加を実装（未設定時は全文タイトルに縮退）。一覧を期限切れ/今日/近日/期限なしにグルーピングするTODAYタブをデフォルトにし、HOMEのKPIも「今日やる件数」に変える。

### 7. モーニングブリーフィング＋AIデイリープラン （L）
その日初回のHOME表示時に今日の予定・期限タスク・未読通知をGeminiが3〜4文に要約するカードを表示（edge-tts読み上げ付き）。「今日を計画する」ボタンで時間帯付き1日プランを生成し、編集・取捨選択してagendaへ一括登録。夕方は未消化分の翌日繰り越しも提案する。

### 8. サーバーサイド定期実行基盤（cronトリガー＋Autopilot背景実行） （L）
外部無料cron（cron-job.org/GitHub Actions）が/api/cron/tickを叩く方式で、BOARDフローのスケジュールトリガー（日本語プリセット付きUI）とAutopilot AUTO-RUNをブラウザ非依存化。Render freeのスリープと共存しつつ「毎朝ニュース要約→通知」「寝ている間にミッション完走」が本当に動く。

### 9. 実データ接続ステップ（fetch/HTTP・Webhook受信・条件分岐） （L）
URL本文抽出ステップ・任意REST呼び出しステップ・IF/AI判定分岐を追加し、各フローにHMAC付きWebhook受信URLを発行。「記事を取得→要約→重要なときだけ通知」など、AI生成のみだった自動化が外部の実データを扱えるようになる。

### 10. Web Push通知＋ワンタップ承認 （L）
VAPID自己生成＋pywebpushで承認待ち・ミッション完了・朝ブリーフィングをアプリを閉じていてもロック画面へ配信（iOS 16.4+のPWA対応）。通知内のHMAC署名リンクでアプリを開かずに承認/却下でき、Income運用が通知1タップで完結する。

## 🧱 基盤整備 — 信頼性・運用（8件）

### 1. Service Workerオフラインシェル＋更新トースト （M）
アプリシェルをprecacheしGET系APIをstale-while-revalidateで返すSWを導入、圏外でもHUDが即表示される。あわせてupdatefound/waiting検知で「SYSTEM UPDATE READY」トースト→skipWaiting再読込を実装し、古いキャッシュ残留問題も同時に潰す。Web Push・バッジ等の前提基盤。

### 2. Gemini利用量メーター（無料枠ガード） （M）
FastAPI側でGemini呼び出しをラップしusage_metadataをSupabaseに記録、DIAGNOSTICSに本日のRPM/RPD/トークン量vs無料枠上限のゲージと閾値警告を表示。無料枠429での突然死を事前に可視化する。

### 3. ワンクリック全データバックアップ/復元 （M）
tasks・agenda・vault・studio・automations・missions・memoryを1つのJSONに集約する/export/allと復元用/import/allを追加し、Keychain付近にBACKUPボタンを配置。Supabase freeの90日無操作停止やデータ事故に備えた可搬性を確保する。

### 4. 統合実行履歴ログ＋失敗リトライ （M）
BOARDフロー・Studioワークフロー・夜間生成・Autopilotの全実行をautomation_runsテーブルに記録し、n8nのExecutions風に入出力・エラー・所要時間をタイムライン表示、失敗はワンクリック再実行。2ドメインで重複していた案を統合し「失敗が見えない」自作自動化の最大の弱点を解消する。

### 5. 記憶パネル（長期記憶の可視化・管理） （M）
バックエンドで既に稼働中のagent_memoryに一覧・削除・ピン留めのCRUDとUIを追加し、「AIが覚えていること」を設定から管理可能に。チャット中の記憶更新時はバブル下にチップ表示し、透明性と信頼を担保する。

### 6. コネクタステータスボード （M）
YouTube/Shutterstock/note/Leonardo等のキー設定状況を「未設定/設定済み/接続OK」でカード表示し、読み取り専用APIでの接続テストと媒体別有効化トグルを提供。publish-approvedは有効媒体のみ実行し、配信失敗の原因切り分けが即できる。

### 7. フロービルダーの並び替え・作成後編集 （M）
BOARD/Studioのステップを依存導入済みのframer-motion Reorderでドラッグ並び替え・インライン編集可能にし、PATCHエンドポイントを1本追加。「作成後は作り直すしかない」現状を解消する自動化編集の基本体験。

### 8. Vault資料一覧＋ノートブック管理 （M）
選択中ノートブックの資料タイトル一覧・個別閲覧・削除・リネームと、ノートブック自体のリネーム/削除（確認ダイアログ付き）を追加。誤投入した資料が確認も削除もできず回答を汚染し続ける衛生問題を解消する。

## 🔒 先行必須（批判レビューで検出された欠落 — 特にセキュリティ/モバイル）

### 1. API認可の実効化（公開バンドル内トークンの廃止＋レート制限） 
現状は NEXT_PUBLIC_API_TOKEN が JS バンドルに埋め込まれ、Vercel の URL を開いた人全員に FastAPI への全権 Bearer トークンが渡る（EntryGate の PIN もクライアント側の見せかけ）。既に導入済みの Supabase Auth の JWT を FastAPI 側で検証する方式（JWKS 検証は無料・依存追加のみ）へ移行し、CORS の allow_origins="*" フォールバックを自ドメイン固定に、slowapi で簡易レート制限を追加する。Web Push の署名リンクや cron エンドポイントを公開する前の必須前提だが、ロードマップにはどこにも含まれていない。

### 2. 間接プロンプトインジェクション防御（外部コンテンツ×自動実行のガードレール） 
Big Bets の「URL/PDF取込」「fetch/HTTPステップ」「Webhook受信」「サーバーサイド定期実行」は、取り込んだ外部テキスト内の指示を Gemini が実行してしまう間接インジェクションの入口をまとめて開く。取込テキストは『資料データであり指示ではない』とプロンプト上で隔離し、自動化フロー内の HTTP 呼び出し先を許可ドメインリスト制、公開・通知など副作用ステップは外部データ経由の実行時に承認キューを必ず経由させる、の3点をデータ接続系機能の設計原則として先に決めておくべき。

### 3. モバイル片手操作の最適化（サムゾーンUI・キーボード回避） 
スマホ PWA が主戦場のはずだが、下部タブナビ（モード切替を親指圏内に）、44px 以上のタッチターゲット、チャット入力の visualViewport 追従（ソフトキーボードで入力欄が隠れる問題）、スワイプでの履歴パネル開閉、ホーム画面インストール誘導（beforeinstallprompt / iOS 手順シート）といった片手操作の観点が全カテゴリに存在しない。3D演出やデスクトップ体験の案は厚い一方、毎日ポケットから出して使う導線の設計が丸ごと抜けている。

### 4. 429/障害時の自動フォールバック＆リトライ戦略 
「Gemini利用量メーター」は可視化のみで、実際に無料枠 429 やタイムアウトが起きた瞬間の挙動が未定義。呼び出しラッパーに指数バックオフ付きリトライ、flash→flash-lite へのモデル自動格下げ、チャットには「混雑中・再試行」バブル、夜間バッチには翌スロットへの自動繰延を実装する。定期実行・Autopilot 背景実行を導入すると API 消費が跳ね上がるため、可視化とセットで縮退動作が無いと『朝起きたら全部失敗していた』が常態化する。

### 5. ストレージ容量ガバナンス（localStorage枯渇・Supabase無料枠） 
会話履歴は画像添付を base64 data URL ごと localStorage に保存しており（Chat.tsx）、数枚の写真で 5MB 上限に達し setItem が例外を吐いて履歴保存が静かに壊れる。画像の canvas 縮小・古い会話からの画像間引き・保存失敗時の通知を入れ、あわせて DIAGNOSTICS に localStorage 使用率と Supabase free（DB 500MB）の残量を表示する。Vault の PDF 取込や履歴同期を進めるほど効いてくる、バックアップ案とは別軸の『枯渇させない』設計。

### 6. エラー可観測性（フロント/バックの障害を1か所で見える化） 
統合実行履歴ログは自動化ジョブ専用で、フロントの未捕捉例外・SSE切断・API 5xx を後から追う手段がない。window.onerror / unhandledrejection を拾って FastAPI の軽量 /api/client-log に送り Supabase に貯める自前収集（または Sentry 無料枠）を導入し、DIAGNOSTICS に直近エラー一覧を表示する。開発者=唯一のユーザーである本アプリでは『スマホで再現した不具合を PC で調べられる』ことが開発速度に直結するのに、この足場が計画に無い。

> **講評**: コスト可視化・データ可搬性・音声ファースト・習慣化（朝ブリーフィング＋Push）は既にカバーされた完成度の高いロードマップだが、「セキュリティの実効性（バンドル露出トークン・インジェクション）」と「モバイル片手操作」という毎日使うJARVISの土台2本柱が丸ごと欠落しており、外部データ接続・定期実行系のBig Betsを進める前に先行着手すべき。
